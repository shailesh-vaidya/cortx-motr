Fault Injection Service: objectives and use

Fault Injection Service is intended to provide fault injection functionality at run time. To be enabled the service must appear in configuration database with service type M0_CST_FIS under the process which is the subject for fault injection. Additionally, command line parameter '-j' must be specified for the process to unlock the service start. This "double-lock" mechanism is intended to protect cluster from accidental FIS start.

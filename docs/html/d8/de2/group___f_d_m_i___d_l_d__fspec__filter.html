Detailed Description

FDMI Functional Specification
FDMI filter evaluator description

In-memory representation of FDMI filter expression.

FDMI filter expression has a tree structure. Tree nodes have type m0_fdmi_flt_node.

There are three types of nodes:

  • constant operand (m0_fdmi_flt_operand);
  • variable operand (m0_fdmi_flt_var_node). Its value should be evaluated in run-time using the information contained in the tree node. Usually it is value of some field in FDMI record;
  • operation (m0_fdmi_flt_node). Contains operation code and links to the operands for this operation.

Every operand has a type (m0_fdmi_flt_operand_type). Types of operands should be checked during operation execution.

No special functions for tree traversal are provided. User of the filter expression should use m0_fdmi_flt_op_node::opnds field for tree traversal.

◆ m0_fdmi_flt_op_code

enum m0_fdmi_flt_op_code

Filter operation code

Specifies what operation should be applied to operands. There are two types of operations:

  • common operations
  • specific to some FDMI record types

Common operations are implemented by FDMI filter evaluator (FDMI filter evaluator description), implementation of specific operations should be provided for FDMI filter evaluator using m0_fdmi_eval_add_op_cb.
